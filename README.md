# LAN Cookie Sync | 局域网 Cookie 同步工具

本项目专为解决“素材站单账号只能登一台电脑”的限制而设计。通过局域网中转，将已登录电脑的 Cookie 实时同步给其他电脑。

## 项目结构

-   `server/`: 中转服务器，运行在局域网内任意一台电脑上。
-   `extension/`: Edge/Chrome 浏览器插件，安装在所有需要同步的电脑上。

---

## 快速开始

### 1. 启动中转服务器 (在任意一台电脑运行)

你可以选择运行 **Node.js** 或 **Python** 版本。

#### 选项 A: Node.js (推荐)
1. 进入 `server` 目录：`cd server`
2. 安装依赖：`npm install`
3. 启动：`npm start`

#### 选项 B: Python
1. 进入 `server` 目录：`cd server`
2. 安装依赖：`pip install fastapi uvicorn websockets`
3. 启动：`python relay.py`

*注意：请记住控制台显示的局域网 IP 地址（例如 `192.168.1.5:3000`）。*

### 2. 安装 Edge/Chrome 插件

1. 打开 Edge 浏览器，进入 `edge://extensions/`。
2. 开启左下角的 **“开发人员模式”**。
3. 点击 **“加载解压的扩展”**，选择项目中的 `extension` 文件夹。
4. 在所有参与同步的电脑上都执行此操作。

### 3. 操作流程

#### A. 发送端 (已登录账号的电脑)
1. 点击插件图标，点击 **“发送端”** 标签。
2. 中转站地址输入：服务器的 IP:端口 (例: `192.168.1.5:3000`)。
3. 目标域名输入：素材站的域名 (例: `699pic.com`)。
4. 点击 **“立即推送同步”**。

#### B. 接收端 (局域网其他电脑)
1. 点击插件图标，点击 **“接收端”** 标签。
2. 中转站地址输入：同样的服务器 IP:端口。
3. 点击 **“开始监听”** (状态变为“已连接”)。
4. 当发送端同步后，接收端会收到系统通知，此时刷新对应网页即可发现已变为登录状态。

---

## 注意事项

-   **安全性**：Cookie 相当于你的账号密码，请确保仅在受信任的局域网内使用，不要将服务器暴露在公网。
-   **域名**：输入的域名需尽量准确（如 `abc.com` 会匹配该域名下的所有 Cookie）。
-   **手动触发**：为了防止死循环，本工具采用手动推送逻辑。当你发现接收端掉线或失效时，在发送端再次点击推送即可。

---

### FAQ
- **无法连接服务器？** 检查防火墙是否允许 3000 端口通信。
- **Cookie 没生效？** 尝试清除该站点的 Cookie 后，再次从发送端同步。
